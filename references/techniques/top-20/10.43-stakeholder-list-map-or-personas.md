# Stakeholder List, Map, or Personas (BABOK 10.43)

## Overview

Stakeholder List, Map, or Personas helps identify, analyze, and manage stakeholders who affect or are affected by a proposed solution. It models stakeholder characteristics across three core tools:
- **Stakeholder Register (List):** Directory of key participants, roles, and project domains.
- **RACI Matrix (Map):** Visual chart mapping authorities (Responsible, Accountable, Consulted, Informed) for project deliverables.
- **User Personas:** Semi-fictional representations of target users based on real data, outlining their goals, behaviors, and frustrations.

## When to Use

- User needs to establish who should participate in elicitation sessions and sign off on requirements.
- User needs to design a system's target user experience (UX) based on end-user characteristics.
- Task 3.2 (Plan Stakeholder Engagement), Task 4.2 (Conduct Elicitation), Task 5.1 (Prepare for Elicitation), Task 7.1 (Specify and Model Requirements).

## Do not use when
- Specifying database storage schemas (use Data Dictionary 10.12 instead).
- Designing step-by-step sequential workflows (use Process Modelling 10.35 instead).

## Step-by-Step for AI Agent

1. **Verify Context Inputs:** Scan for lists of company roles, external customers, executive sponsors, and system deliverables.
2. **⚠️ SHARP NEGATIVE CONSTRAINT:** If the target audience or the core solution deliverables are completely unknown, STOP. Request the project charter or high-level business goals. You must not invent stakeholder names or RACI responsibilities (like making a "Customer" Accountable for system architecture) without input facts.
3. **Map the RACI Matrix:** Ensure that for every deliverable, exactly **one** role is assigned **Accountable (A)**. Multiple roles can be Responsible (R), Consulted (C), or Informed (I).
4. **Draft User Personas:** Synthesize user motivations, technical skills, core goals, and frustrations from elicitation notes.
5. **Generate Output:** Populate the standard template below.

## Common Mistakes & CBAP Gotchas

> **⚠️ AI AGENT INSTRUCTION:** If any of these traps are detected in the user's request, surface a `⚠️ CBAP Gotcha` alert before generating the output.

**Trap 1 — Multiple "A"s in a RACI Row**
- Wrong: Assigning "A" (Accountable) to both the Product Owner and the Project Sponsor for the same deliverable. (If everyone is accountable, no one is).
- Right: Enforce exactly one "A" per row to establish clear decision authority.

**Trap 2 — Designing "Fictional" Personas with No System Relevance**
- Wrong: Writing that a persona *"loves playing golf and drinking coffee"* when designing a backend database schema for inventory control.
- Right: Focus strictly on demographic, psychographic, and operational traits that influence system usage and requirements.
