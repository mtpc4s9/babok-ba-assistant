# Business Cases (BABOK 10.07)

## Overview

A Business Case provides a formal justification for a proposed enterprise investment or change strategy. It compares the expected benefits and value against the estimated costs, effort, and risks of multiple options, including the baseline "Do Nothing" scenario, to guide C-suite decision-making.

## When to Use

- User needs to secure executive approval or budget allocation for a proposed change.
- User needs to compare the financial and strategic viability of different solution options.
- User needs to document the cost-benefit analysis and risk profile of a planned investment.
- Primarily used in Task 6.4 (Define Change Strategy) and Task 7.6 (Analyze Potential Value).

## Do not use when
- The solution scope is completely undefined and no cost or benefit estimates can be made.
- The work is an absolute regulatory mandate where no alternatives or value trade-offs exist (though a light business case can still help trace execution costs).

## Quick Reference

| Decision Point | Use This |
| :--- | :--- |
| Assessing baseline comparison | **"Do Nothing" Scenario** (always mandatory) |
| Multiple solution options | **Comparative Financial Analysis** (NPV, ROI, Payback) |
| Intangible impacts | **Qualitative Benefits Section** (Strategic Alignment) |
| Primary KAs | KA03, KA05, KA06, KA07, KA08 |
| Most common tasks | Task 6.4 (Change Strategy), Task 7.6 (Potential Value) |

## Step-by-Step for AI Agent

1. **Scan Input Context:** Verify if raw cost structures, strategic goals, and benefit assumptions are provided.
2. **⚠️ SHARP NEGATIVE CONSTRAINT:** If costs, benefits, or strategic alignment information is missing or highly speculative, STOP. Request the user to provide estimated values or assumptions before proceeding. Never invent financial data.
3. **Analyze the "Do Nothing" Option:** Formulate a strict operational and financial impact statement of taking no action.
4. **Structure the Comparative Matrix:** Outline the costs (Capex/Opex), benefits (tangible/intangible), and risks for each viable option.
5. **Compute Metrics (if data is available):** Calculate basic ROI, Net Present Value (NPV), or Payback Period, clearly calling out assumptions.
6. **Generate Output:** Populate the standard template below.

## 🤖 AI AGENT INSTRUCTION (EXECUTION RULES)
- **Financial Rigor:** Always present costs and benefits as distinct entities (do not mix initial capital expenditure with ongoing operational costs).
- **"Do Nothing" Imperative:** Never omit the "Do Nothing" alternative. It is the primary control group against which all changes must be measured.
- **Traceability:** Every figure used in the Cost-Benefit Analysis must be linked to its source or clearly marked as a calculated assumption.

## Common Mistakes & CBAP Gotchas

> **⚠️ AI AGENT INSTRUCTION:** If any of these traps are detected in the user's request, surface a `⚠️ CBAP Gotcha` alert before generating the output.

**Trap 1 — Omitting the "Do Nothing" Baseline**
- Wrong: Comparing only Option A vs. Option B.
- Right: Always include Option 0 (Maintain Status Quo) to justify the investment necessity.

**Trap 2 — Double-Counting Benefits**
- Wrong: Adding time savings of employees as direct revenue while also counting reduced operational cost.
- Right: Separate direct hard savings (e.g., license reductions) from soft productivity gains (e.g., hours saved).

**Trap 3 — Unrealistic Payback Windows**
- Wrong: Claiming a massive enterprise ERP migration will pay for itself in 6 months.
- Right: Validate payback periods against industry standards (usually 18 to 36 months for large-scale changes).
