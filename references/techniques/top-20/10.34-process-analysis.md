### Process Analysis (BABOK 10.34)

#### Overview
Process analysis assesses a process for its efficiency and effectiveness, as well as its ability to identify opportunities for change [1]. It evaluates how work is performed to deliver value across multiple functional areas [2]. Common changes include reducing time, modifying hand-offs to remove errors (bottlenecks), and increasing automation [8].

#### When to Use
*  User needs to recommend a more efficient or effective process [1].
*  User needs to determine the gaps between the current (as-is) and future state (to-be) of a process [1].
*  User needs to select a process improvement framework (e.g., Lean, Six Sigma, SIPOC, Value Stream Mapping) [8].
*  **Primary Tasks:** 3.5 (Identify Business Analysis Performance Improvements), 6.1 (Analyze Current State), 8.4 (Assess Enterprise Limitations), 8.5 (Recommend Actions to Increase Solution Value) [9].

#### Do not use when
*  The primary goal is simply to visually map or document how work is carried out without analyzing its performance (Use **10.35 Process Modelling** instead) [3].
*  The process relies heavily on unpredictable, knowledge-based, or decision-intensive human tasks where standard analysis may prove ineffective [10].

**🤖 AI AGENT INSTRUCTION (PROCESS ADVISORY INTERVIEW - ZERO HALLUCINATION):** 
Because a single process can be improved in multiple ways (e.g., IT automation, human resource reallocation, step elimination), you MUST NOT assume the solution. When the user requests a Process Analysis, HALT execution and conduct this 4-question interview:

*"To generate the most viable improvement options for this process, I need to understand your strategic drivers. Please clarify:*
1. *What is the **Primary Pain Point**? (e.g., Takes too long, costs too much, too many errors/rework).*
2. *What is the **Root Cause** of this pain point based on your current understanding?*
3. *What is our **Improvement Focus**? (e.g., **Lean** to eliminate waste/wait times, **Six Sigma** to reduce errors/variation, or **IT-centric** to automate manual steps).*
4. *Are there any **Constraints**? (e.g., "Zero IT budget, only operational changes allowed")."*

Do NOT proceed to generate the evaluation matrix below until the user provides this context.
