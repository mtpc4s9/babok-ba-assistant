# Business Model Canvas (BABOK 10.08)

## Overview

The Business Model Canvas (BMC) is a strategic management template for developing new or documenting existing business models. It visualizes the flow of value creation, distribution, and capture across nine building blocks: Key Partners, Key Activities, Key Resources, Value Propositions, Customer Relationships, Channels, Customer Segments, Cost Structure, and Revenue Streams.

## When to Use

- User needs to design a new venture, business line, or solution offering.
- User (especially a Product Owner) needs to understand how a system capability aligns with the overall business model.
- User needs to analyze competitor models or align stakeholder views on business operations.
- Task 6.2 (Define Future State), Task 7.4 (Define Requirements Architecture), Task 7.6 (Analyze Potential Value).

## Do not use when
- Detailed, step-by-step operational workflows or system integrations are needed (use Process Modelling or Interface Analysis instead).
- Granular, line-item financial projections or cash-flow forecasts are required (use Business Cases or Financial Analysis instead).

## Quick Reference

| Decision Point | Use This |
| :--- | :--- |
| Core value proposition missing | Stop and brainstorm **Value Propositions** block first |
| Target audience undefined | Stop and define **Customer Segments** |
| High-level operational alignment needed | **Business Model Canvas** |
| Primary KAs | KA06, KA07 |
| Most common tasks | Task 6.2 (Future State), Task 7.6 (Potential Value) |

## Step-by-Step for AI Agent

1. **Verify Context Inputs:** Scan for details about the product, market, partners, costs, and revenues.
2. **⚠️ SHARP NEGATIVE CONSTRAINT:** If the "Value Propositions" (what problem is being solved) or "Customer Segments" (who it is being solved for) are unknown or completely blank, STOP. Request the user to clarify the core value and audience before proceeding. A canvas without a value proposition is invalid.
3. **Map the Nine Blocks:** Structure the information logically, starting from the value proposition outward to the operational blocks (partners, activities, resources) and customer blocks (relationships, segments, channels), ending with the financial blocks (costs, revenues).
4. **Generate Output:** Present the standard layout below.

## 🤖 AI AGENT INSTRUCTION (EXECUTION RULES)
- **Logical Alignment:** Ensure the Key Activities, Key Resources, and Key Partners directly support the Value Propositions. Ensure the Channels and Customer Relationships directly connect to the target Customer Segments.
- **Conciseness:** Keep bullet points under each block concise, precise, and business-focused. Avoid long narrative paragraphs.

- **Awareness/Delivery:** [INSERT: Bullet points | How the enterprise communicates with and reaches customer segments to deliver the value | System Interfaces]

## Common Mistakes & CBAP Gotchas

> **⚠️ AI AGENT INSTRUCTION:** If any of these traps are detected in the user's request, surface a `⚠️ CBAP Gotcha` alert before generating the output.

**Trap 1 — Disconnected Value Proposition and Customer Segment**
- Wrong: Describing a value proposition (e.g., "High-speed algorithmic trading API") that has no alignment with the listed customer segment (e.g., "Non-technical casual investors").
- Right: Maintain strict logical alignment between who you serve and what value you offer.

**Trap 2 — Treating the Canvas as a Task List**
- Wrong: Listing temporary, project-level tasks (e.g., "Set up daily stand-ups", "Hire 3 developers") under Key Activities.
- Right: Key Activities must reflect the ongoing, core operational processes required to run the business model at scale (e.g., "Continuous API optimization", "SLA maintenance").
