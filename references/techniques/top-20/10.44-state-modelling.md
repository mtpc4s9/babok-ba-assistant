### State Modelling (BABOK 10.44)

#### Overview
State modelling is used to describe and analyze the different possible states of an entity within a system, how that entity changes from one state to another, and what can happen to the entity when it is in each state. It provides a complementary view to process models: instead of showing a flow of work, it shows what happens to *one specific entity* across all the processes that affect it.

#### When to Use
*   User needs to understand the complete life cycle of a complex business object (e.g., Order, Contract, Invoice, User Account).
*   User needs to identify missing business rules or validation checks during state transitions.
*   **Primary Tasks:** 7.1 (Specify and Model Requirements).

#### Do not use when
*   User needs to show the sequential flow of work across multiple roles or departments (Use **10.35 Process Modelling** instead).
*   The entity is too simple and does not have complex behaviours or transition rules.
